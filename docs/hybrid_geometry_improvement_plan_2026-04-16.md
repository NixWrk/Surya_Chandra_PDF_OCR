# План доработок гибридной геометрии (2026-04-16)

## Цель

Повысить точность совпадения невидимого текстового слоя с реальной геометрией букв на странице PDF в режиме:

- `chandra` text + `surya` geometry

без потери качества распознавания текста `chandra`.

## Что уже видно по текущему коду и логам

1. В `auto` выбор кандидата переоптимизирован под текстовую похожесть, а не под геометрию страницы.
2. В `softline` текущая логика фактически форсирует `primary` и уже потом смешивает по Y.
3. На полном прогоне `full_hybrid_eval_20260414_163835`:
   - `book_handwritten`: в `auto` выбрано `secondary` на 30/33 страницах.
   - `gost`: в `auto` выбрано `secondary` на 35/37 страницах.
4. Используется line-level геометрия + эвристическое деление строки по длине токенов; это ограничивает точность на уровне букв/слов.
5. В Surya-плагине доступны `words` с bbox, но в sidecar для гибридной сборки сохраняются только `text_lines`.

## План работ

## 1) Метрика качества геометрии (обязательно сначала)

1. Добавить отдельную метрику геометрии, не зависящую от текстовой похожести:
   - overlap/IoU с детектированными line-boxes страницы;
   - средняя ошибка центра/базовой линии.
2. Добавить сводный отчёт по документу и по страницам для A/B/C policy.

Критерий готовности: есть стабильный числовой KPI геометрии для регрессионных прогонов.

## 2) Логика выбора кандидата (`auto`/`softline`)

1. Пересобрать `score`: уменьшить вес чисто текстовых признаков, добавить геометрический fit к странице.
2. Исправить `softline`: выбирать `primary`/`secondary` по реальному score, а не по жёсткому приоритету.
3. Добавить режим `surya_prefer`: Surya как default, fallback только при явном провале.

Критерий готовности: `auto` перестаёт системно «скатываться» в Chandra-геометрию, если Surya объективно ближе к странице.

## 3) Повышение точности placement

1. Перейти в гибриде с line-level на word-level Surya geometry (использовать `words` bbox).
2. Для unmatched-токенов заменить простую интерполяцию на локально-монотонное выравнивание по строкам.
3. Улучшить базовую формулу baseline/font fit (калибровка по ascent/descent шрифта).

Критерий готовности: снижение визуальных сдвигов токенов на сложных страницах (плотный текст, таблицы, неровные строки).

## 4) Валидация и защита от регрессий

1. Добавить тесты на:
   - выбор источника геометрии по policy;
   - корректность blending;
   - использование word-level bbox в гибриде.
2. Зафиксировать acceptance-gate для merge:
   - геометрический KPI улучшается;
   - извлекаемый текст не деградирует.

## Порядок внедрения

1. Сначала метрика (раздел 1).
2. Затем scoring/policy (раздел 2).
3. Затем word-level placement и baseline-калибровка (раздел 3).
4. После каждого шага — прогон на `gost + book_handwritten` и фиксация отчёта.

## Ожидаемый эффект

1. Быстрый эффект: правка scoring/policy.
2. Максимальный прирост точности «по буквам»: переход на word-level Surya geometry.
3. Стабильность в долгую: метрика + тесты + acceptance-gate.
